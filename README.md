# 📊 Week 5: Exploratory Data Analysis (EDA)

## 📌 Overview
This project focuses on performing Exploratory Data Analysis (EDA) on a student performance dataset. The goal is to uncover patterns, trends, and relationships in learner data to support academic decision-making and improve future interventions.

The analysis is conducted using Python (Pandas), with results summarized in Excel for an executive-ready report.

---

## 🎯 Objectives
- Assess data quality and readiness
- Perform descriptive statistical analysis
- Explore data distributions and identify outliers
- Understand relationships between variables
- Generate insights into student performance

---

## 📂 Dataset
- File: `Week-5-Student-Scores.csv`
- Type: Structured data (tabular format)
- Example fields:
  - Student_ID
  - Gender
  - Subject
  - Study_Hours_per_Week
  - Attendance_Rate
  - Previous_Score
  - Quiz Scores (Quiz1, Quiz2, Quiz3)
  - Final_Score

---

## 🛠️ Tools & Technologies
- Python (Pandas, NumPy, Matplotlib/Seaborn)
- Jupyter Notebook
- Microsoft Excel (for summary reporting)

---

## 📊 Project Tasks

### 🔹 Q1: Data Readiness & Quality Checks
- Loaded dataset using Pandas
- Assessed:
  - Dataset shape (rows & columns)
  - Data types
  - Missing values per column
  - Duplicate records
- Documented data cleaning decisions:
  - Handling missing values (drop/impute)
  - Removing duplicates

📌 Deliverables:
- Diagnostics summary table
- 3–5 bullet points explaining data cleaning decisions

---

### 🔹 Q2: Descriptive Statistics & Distributions
- Generated descriptive statistics:
  - Mean, Median, Standard Deviation
  - Min, Max
  - Percentiles (25th, 50th, 75th)
- Visualized distributions using:
  - Histograms
  - Boxplots

📊 Key variables analyzed:
- Final_Score
- Study_Hours_per_Week
- Attendance_Rate

📌 Insights:
- Identified skewness and spread of data
- Detected potential outliers
- Interpreted implications for student performance
